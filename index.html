/ DUAL_ENGINE.SYS

El sistema provee dos motores especializados. Selecciona el vector de aproximación correcto basándote en el modelo LLM y el radio de impacto de la tarea.

@core

CORE ENGINE

"Think deeply, act once."

  • » Contexto: Denso / Máximo disponible.
  • » Estrategia: Planeación y delegación.
  • » Caso de uso: Tareas multi-paso complejas.
  • » LLM Ideal: MiniMax, modelos o1, y APIs facturadas por request.
  • » Economía: Exprime al máximo las ventanas de contexto gigantes. Al no importar el peso en tokens, es ideal si pagas una tarifa por petición.
REGLA: Nunca escribe código directamente. Analiza la totalidad del repositorio usando get_active_brain() y lanza sub-agentes para la ejecución.
@particle

PARTICLE ENGINE

"Precision over volume."

  • » Contexto: Mínimo / Divulgación progresiva.
  • » Estrategia: Edición AST función por función.
  • » Caso de uso: Fixes rápidos, refactoring atómico.
  • » LLM Ideal: Flash, Haiku y modelos facturados por token.
  • » Economía: Minimiza el gasto enviando fragmentos quirúrgicos en lugar de archivos enteros, reduciendo el costo por token hasta un 95%.
TOKEN SAVER: ~95% reducción. Usa get_file_skeleton para escanear, read_function para aislar y replace_function para mutar.
